# 28.1 Report API 契约（已冻结）

> 状态：**A1 / B1 / C1 / D1 / E1 均已确认**（2026-05-24 拍板）。宇宙页已按本契约接入；`workbench` 历史/报告页不在 E1 范围内。

## 目标

让 **session / report / history / report-page** 共用后端真实 report 对象，而不是仅前端 `localStorage` 拼装。

## 数据对象

与 `backend/src/types/report.ts` 中 `StoredInterviewReportSummary` 一致。

与前端 `PersistedReportSummary` 对齐字段：

- `id`, `sessionId`, `topic`, `source`
- `weaknessTags`, `answeredCount`, `totalCount`
- `summaryHeadline`, `summaryBody`
- `sourceDocumentId`, `sourceDocumentName`
- `practicePlan`（可选）

## 接口

### `GET /api/interview/reports`

列表，按 `updatedAt` 倒序。

```json
{
  "reports": [
    {
      "id": "report-xxx",
      "sessionId": "xxx",
      "topic": "vue",
      "summaryHeadline": "...",
      "answeredCount": 2,
      "totalCount": 3,
      "weaknessTags": [],
      "createdAt": "ISO",
      "updatedAt": "ISO"
    }
  ]
}
```

### `GET /api/interview/reports/:sessionId`

一轮模拟一条报告，按 `sessionId` 读取。

```json
{
  "report": { /* StoredInterviewReportSummary */ }
}
```

404：`REPORT_NOT_FOUND`

### `POST /api/interview/reports/generate`

结束本轮时调用。根据该 `sessionId` 下已存储的 thread 会话消息，生成最小 summary 并落盘。

请求体（最小）：

```json
{
  "sessionId": "required",
  "topic": "optional",
  "source": "optional",
  "answeredCount": 0,
  "totalCount": 0,
  "weaknessTags": []
}
```

响应：

```json
{
  "report": { /* StoredInterviewReportSummary */ },
  "created": true
}
```

若已存在同 `sessionId` 报告：默认 **覆盖更新**（`created: false`），避免重复结束产生多条。

## 存储

- 文件：`backend/data/interview-reports.json`
- 结构：`{ "reports": StoredInterviewReportSummary[] }`
- 写法：与 `interview-sessions.json` 相同（tmp + rename）

## 前端消费顺序（宇宙页）

1. 报告列表 / 总览：**优先** `GET /reports`，与 `localStorage` 按 `sessionId` 合并，服务端覆盖同 id 本地项。
2. 单条摘要：合并列表未命中时 **`GET /reports/:sessionId`**（`useMockInterviewSpaceReportHydration`）。
3. 详情证据：摘要无 `answerSnapshot` 时 **`GET /sessions/:sessionId/:threadId`** 补消息摘录。
4. 后端不可用或 404：**fallback** `localStorage` + `buildReportSummary`。
