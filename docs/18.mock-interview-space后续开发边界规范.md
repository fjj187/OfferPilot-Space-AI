# 18.mock-interview-space 后续开发边界规范

## 1. 核心原则

`mock-interview-space.vue`（模拟面试星际页面主文件）只允许承担“页面装配”和“场景编排”职责，不再承接新的业务 UI（用户界面）、业务状态、场景样式或动画细节。

后续任何新需求，必须先判断它属于哪个已拆分边界，再写入对应文件。禁止因为“方便”直接把代码继续塞回 `mock-interview-space.vue`（模拟面试星际页面主文件）。

如果用户提到“宇宙页面”“星际页面”“轨道页面”“截图里的页面”，默认开发目标是 `/showcase/mock-interview-space`（宇宙模拟面试页），不是 `src/views/workbench`（旧工作台页面目录）。

---

## 1.1 旧工作台页面禁改规则

开发 `/showcase/mock-interview-space`（宇宙模拟面试页）时，默认禁止修改：

1. `src/views/workbench/*`（旧工作台页面文件）。
2. `src/components/workbench/*`（旧工作台组件文件）。
3. `src/composables/useOverviewLaunchState.ts`（旧工作台总览状态组合函数）。
4. `src/composables/useLibraryWorkspaceState.ts`（旧工作台资料库状态组合函数）。
5. `src/views/workbench/practice.vue`（旧工作台专项训练页）。

只有在用户明确说“改工作台页面”“改旧页面”“改 workspace 页面”，或点名上述具体文件时，才允许进入这些文件开发。

如果任务名称和旧页面名称重名，例如“专项训练页面”“资料库”“报告页”，但上下文来自 `/showcase/mock-interview-space`（宇宙模拟面试页）或截图里的宇宙页面，就必须改 `src/components/showcase/mock-interview-space`（宇宙页面组件目录）下的对应场景组件。

---

## 2. 主文件允许做什么

`mock-interview-space.vue`（模拟面试星际页面主文件）只允许保留：

1. 页面根容器。
2. 顶层 `ref`（模板引用）和少量跨组件桥接。
3. `SpaceHeader.vue`（顶部栏组件）、`SpaceVisualStage.vue`（星球舞台组件）、`SpaceOrbitNav.vue`（轨道导航组件）、`SpaceScrollCapsule.vue`（滚动胶囊组件）、`SpaceContentPanel.vue`（内容面板组件）的装配。
4. 场景切换入口，例如 `openSceneContent`（打开场景内容函数）。
5. 跨场景事件分发，例如从总览跳资料库、从报告回模拟面试。
6. 页面级背景、页面根布局、少量跨组件主题变量。

如果新增代码不属于以上范围，默认不应该写进 `mock-interview-space.vue`（模拟面试星际页面主文件）。

---

## 3. 主文件禁止承接什么

禁止继续往 `mock-interview-space.vue`（模拟面试星际页面主文件）添加：

1. 新场景的大段 `template`（模板）。
2. `mock`（模拟面试场景）、`library`（资料库场景）、`overview`（总览场景）、`report`（报告场景）的内部 UI（用户界面）。
3. 表单输入、列表渲染、卡片布局、弹窗、筛选器等业务组件。
4. 模拟面试提交、报告拼装、资料筛选等业务状态。
5. 场景内部 SCSS（样式表）和 `:deep`（Vue 深度选择器）覆盖。
6. `GSAP`（动画库）细节、轨道动画细节、滚动算法细节。
7. 新增常量文案、主题色、轨道配置。

如果确实需要改主文件，改动应保持“小而薄”，并说明为什么不能放到已有子组件或 `composable`（组合函数）里。

---

## 4. 新需求应该放哪里

### 4.1 内容面板和场景分发

放到 `SpaceContentPanel.vue`（内容面板组件）：

1. 场景组件的挂载顺序。
2. `overview`（总览场景）、`library`（资料库场景）、`mock`（模拟面试场景）、`report`（报告场景）的切换容器。
3. 内容区过渡动画，例如 `panel-swap`（面板切换动画）。
4. 内容区通用壳层样式。

不要把具体业务卡片继续写在 `SpaceContentPanel.vue`（内容面板组件）里。如果内容变复杂，应继续拆子组件。

### 4.2 模拟面试场景

放到 `SpaceMockScene.vue`（模拟面试场景组件）：

1. 题目区。
2. 提示区。
3. 问答消息区。
4. 回答输入区。
5. 模拟面试场景专属样式。

模拟面试业务状态放到 `useMockInterviewSpaceMockState.ts`（模拟面试状态组合函数），不要放回场景组件或主文件。

### 4.3 资料库场景

放到 `SpaceLibraryScene.vue`（资料库场景组件）：

1. 导入入口。
2. 筛选标签。
3. 资料列表。
4. 资料选择事件。
5. 资料库场景专属样式。

资料预览展示优先复用 `DocumentPreviewPanel.vue`（文档预览组件）。如果资料预览逻辑继续变复杂，应拆出资料库专用预览容器，不要塞回主文件。

### 4.4 总览场景

放到 `SpaceOverviewScene.vue`（总览场景组件）：

1. 训练进度。
2. 总览统计。
3. 总览入口按钮。
4. 总览场景专属样式。

总览只负责展示和触发事件，不直接维护其他场景状态。

### 4.5 报告场景

放到 `SpaceReportScene.vue`（报告场景组件）：

1. 报告头部。
2. 能力统计。
3. 弱点标签。
4. 建议列表。
5. 历史训练摘要。
6. 报告场景专属样式。

报告数据拼装放到 `useMockInterviewSpaceReportScene.ts`（报告场景组合函数）。

### 4.6 轨道与自动播放

放到 `useMockInterviewSpaceOrbit.ts`（轨道控制组合函数）：

1. `orbit`（轨道）顺序。
2. `autoplay`（自动播放）。
3. 上一页、下一页、点击轨道切换。
4. 轨道过渡状态。
5. 轨道幽灵节点和复制文案状态。

轨道展示结构放到 `SpaceOrbitNav.vue`（轨道导航组件），不要把轨道 DOM（文档对象模型）写回主文件。

### 4.7 滚动行为

放到 `useMockInterviewSpaceScroll.ts`（滚动控制组合函数）：

1. 内容区滚动。
2. 题目区置顶。
3. `header fade`（顶部栏淡出效果）。
4. 自动滚动中断。
5. 内容区进入/离开判定。

滚动按钮展示放到 `SpaceScrollCapsule.vue`（滚动胶囊组件）。

### 4.8 星球舞台视觉

放到 `SpaceVisualStage.vue`（星球舞台组件）：

1. 星球图层。
2. 视觉层同步。
3. 宇宙舞台 DOM（文档对象模型）。
4. 舞台专属样式。

如果需要改 `GSAP`（动画库）或视觉层同步算法，应先检查是否属于 `SpaceVisualStage.vue`（星球舞台组件）或单独抽新的视觉 `composable`（组合函数）。

### 4.9 场景配置

放到 `mockInterviewSpaceScenes.ts`（模拟面试星际页场景配置文件）：

1. `scenes`（场景配置）。
2. 场景标题、文案、按钮、主题色。
3. 轨道展示基础数据。
4. 场景相关类型。

不要在 `mock-interview-space.vue`（模拟面试星际页面主文件）里新增场景常量。

---

## 5. 新功能落位判断

新增代码前，按下面顺序判断：

1. 是否只是场景文案或主题配置？是就放 `mockInterviewSpaceScenes.ts`（模拟面试星际页场景配置文件）。
2. 是否是某个场景内部展示？是就放对应 `Space*Scene.vue`（场景组件）。
3. 是否是跨场景内容面板装配？是就放 `SpaceContentPanel.vue`（内容面板组件）。
4. 是否是轨道切换或自动播放？是就放 `useMockInterviewSpaceOrbit.ts`（轨道控制组合函数）。
5. 是否是滚动控制？是就放 `useMockInterviewSpaceScroll.ts`（滚动控制组合函数）。
6. 是否是模拟面试业务状态？是就放 `useMockInterviewSpaceMockState.ts`（模拟面试状态组合函数）。
7. 是否是报告数据拼装？是就放 `useMockInterviewSpaceReportScene.ts`（报告场景组合函数）。
8. 是否是纯展示复用组件？是就优先在 `components`（组件目录）下新增小组件。
9. 以上都不是，才允许考虑改 `mock-interview-space.vue`（模拟面试星际页面主文件）。

---

## 6. 文件行数约束

建议约束：

1. `mock-interview-space.vue`（模拟面试星际页面主文件）保持在 `500 ~ 900` 行。
2. 单个 `Space*Scene.vue`（场景组件）保持在 `200 ~ 500` 行。
3. 单个 `composable`（组合函数）保持在 `100 ~ 300` 行。
4. 如果某个场景组件超过 `500` 行，应优先拆出场景内部子组件。
5. 如果主文件超过 `900` 行，必须先做边界复查，不允许继续堆功能。

---

## 7. 提交前检查

每次涉及 `mock-interview-space`（模拟面试星际页）改动，至少执行：

1. `npm run build`（执行生产构建验证）。
2. 对本次改动文件执行 `eslint`（代码规范检查）。
3. 手动检查 `overview`（总览场景）、`library`（资料库场景）、`mock`（模拟面试场景）、`report`（报告场景）是否仍能切换。
4. 手动检查 `orbit`（轨道切换）、`autoplay`（自动播放）、`scroll`（滚动控制）是否没有回归。

如果全项目 `lint`（代码规范检查）或 `stylelint`（样式规范检查）存在历史问题，不能用它掩盖本次新增问题。本次改动文件应尽量保持干净。

---

## 8. 结论

后续开发默认规则：

`mock-interview-space.vue`（模拟面试星际页面主文件）只做装配，不再堆业务。

新增需求必须先找归属组件或 `composable`（组合函数）。如果找不到归属，先拆新组件，再接入主文件。
