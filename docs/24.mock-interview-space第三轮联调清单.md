# 24.mock-interview-space 第三轮联调清单

## 1. 目标

承接 `docs/23.mock-interview-space功能层第一阶段接入开发文档.md`（功能层第一阶段接入开发文档）第三轮，给当前 `mock-interview-space`（模拟面试星球页）提供一份最小可复用的联调基线。

---

## 2. 主链路检查

1. 从 `library`（资料库）选择一份资料进入 `mock`（模拟面试），题目上下文应跟随资料变化。
2. 提交回答后，页面应明确显示当前走的是：
   - 真实 AI 流式
   - 本地模拟流
3. 点击 `结束本轮并查看报告` 后，应生成本轮 `report summary`（报告摘要）。
4. 点击 `查看完整报告` 后，应在当前页面内弹出报告层，而不是跳去独立工作台页面。
5. 点击 `按弱项继续补练` 后，应进入 `feedback`（专项刷题）场景，并承接本轮 `practicePlan`（专项训练计划）。
6. 从 `feedback`（专项刷题）再次开始训练后，应回到 `mock`（模拟面试）场景，且保留当前训练目标。

---

## 3. 刷新恢复检查

1. 刷新页面后，当前场景应尽量恢复到最近一次访问的场景：
   - `overview`
   - `library`
   - `mock`
   - `feedback`
   - `report`
2. 当前 `activeDocument`（资料上下文）应可恢复。
3. 当前 `activeTopic`（训练主题）应可恢复。
4. 当前 `practicePlan`（专项训练计划）应可恢复。
5. 最近一轮 `report summary`（报告摘要）应仍可用于报告页和续练入口。

---

## 4. 异常路径检查

1. 未配置 `VITE_INTERVIEW_SSE_URL`（面试流式接口地址）时，应清晰回退到本地模拟流。
2. 流式生成过程中点击停止，页面不应卡死，也不应破坏当前 session。
3. 真实流式报错时，应能看到 `streamError`（流式错误）反馈。
4. 报告摘要为空时，报告入口页和报告层应使用兜底文案，而不是空白。
5. 当前资料缺失时，报告页来源信息应优先使用摘要里的 `sourceDocumentName`（来源资料名称）回填。
6. 停止生成后，问答区应追加明确系统提示，且当前 session 仍保持可继续状态。
7. 流式失败时，若当前为真实 AI 流式，应明确提示“真实 AI 流式请求失败”，不要只显示模糊报错。
8. 报告摘要为空时，页面应明确提示“当前为阶段性复盘”，避免用户误以为这是一份完整正式报告。

---

## 5. 构建验证

1. `npm run build`（生产构建）持续通过。
2. 本轮新增的报告弹层、场景恢复和报告来源修正不应破坏现有分层：
   - `mock-interview-space.vue`（主页面）继续负责装配
   - 具体报告展示留在独立组件中
   - 报告场景数据整理留在组合函数中
