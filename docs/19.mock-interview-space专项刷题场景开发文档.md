# 19.mock-interview-space 专项刷题场景开发文档

## 1. 背景

`/showcase/mock-interview-space`（宇宙模拟面试页）里已经有 `feedback`（专项刷题场景）轨道节点，但当前内容区仍是通用占位卡片，只显示“专项刷题区”和两条说明文案。

这个场景应该承接 `report`（复盘报告场景）里的弱点标签，成为宇宙训练链路中的“定向补练”节点，而不是跳到 `workspace/practice.vue`（工作台专项训练页）另起一套页面。

---

## 2. 开发目标

本轮只开发宇宙页面内的专项刷题场景：

1. 新增 `SpacePracticeScene.vue`（专项刷题场景组件）。
2. 在 `SpaceContentPanel.vue`（宇宙内容面板组件）里接入 `displayScene.id === 'feedback'`（当前场景为专项刷题）。
3. 使用 `reportWeaknessTags`（报告弱点标签）、`reportPrimaryWeakness`（主要弱点）和 `currentTopicLabel`（当前主题标签）展示当前弱点。
4. 支持选择弱点、专项专区、题型、题数和难度。
5. 选择完成后点击“开始模拟面试”，回到 `mock`（模拟面试场景）进行训练验证。
6. 不改 `workspace/practice.vue`（工作台专项训练页）。

---

## 3. 职责边界

### 3.1 `SpacePracticeScene.vue`（专项刷题场景组件）

负责：

1. 专项刷题场景 UI（用户界面）。
2. 弱点标签展示。
3. 补练队列展示。
4. 训练节奏和完成标准展示。
5. 场景内样式。

### 3.2 `SpaceContentPanel.vue`（宇宙内容面板组件）

只负责挂载 `SpacePracticeScene.vue`（专项刷题场景组件），并传入必要 `props`（组件属性）。

### 3.3 `mock-interview-space.vue`（模拟面试星际页面主文件）

不新增专项刷题 UI（用户界面），只保持页面装配和场景编排。

---

## 4. 交互设计

专项刷题场景分为四块：

1. 查看当前弱点：展示当前最需要收敛的弱点标签，并允许用户选择一个弱点。
2. 选择专项专区：例如 Vue、JavaScript、TypeScript、工程化、性能优化。
3. 选择题型：例如概念理解题、代码分析题、场景追问题。
4. 设置题数和难度：例如 5 / 10 / 15 题，基础 / 进阶 / 高阶。
5. 开始模拟面试：进入 `mock`（模拟面试场景）完成训练验证。

---

## 5. 验收标准

1. 切换到“专项刷题”轨道节点后，不再显示通用占位卡片。
2. 页面能展示当前主题和当前弱点标签。
3. 用户能选择弱点、专项专区、题型、题数和难度。
4. 点击“开始模拟面试”后进入 `mock`（模拟面试场景）。
5. 没有报告弱点时，能展示默认弱点标签。
6. `npm run build`（执行生产构建验证）通过。
7. 本次改动不触碰 `workspace/practice.vue`（工作台专项训练页）的业务实现。
