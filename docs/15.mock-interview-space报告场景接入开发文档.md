# 15.mock-interview-space 报告场景接入开发文档

这份文档用于承接宇宙页下方“报告场景”的下一阶段开发。

前面几轮已经基本把两件事铺好了：

- 资料库场景开始像真实资料工作区
- 模拟面试场景开始像真实训练入口

但整条主链路还差最后一个关键落点：

**训练结束之后，用户要在宇宙页里看到真实报告，而不是只跳去旧页面看结果。**

---

## 这次开发要解决什么

这次的核心不是再做一个“报告介绍页”，而是把：

**真实训练结果**

接进宇宙页下方的报告场景。

也就是说，这一轮要解决的是：

- 宇宙页里的报告场景，不再只是氛围化占位
- 它要能承接真实 session 和 report summary
- 用户从模拟面试结束，或者从总览 / 资料库点“查看报告”后，能在宇宙页里看到当前这轮训练的真实复盘信息

---

## 为什么现在该优先做报告场景

当前宇宙页主链路已经越来越清楚：

- 总览负责启动
- 资料库负责资料承接
- 模拟面试负责训练过程

但如果报告场景还只是壳子，那这条链路实际上只完成了一半。

用户会有一种感受：

- 我能开始训练
- 我能切资料
- 我也能进入 mock
- 但训练结束后的结果，还没有在同一套宇宙体验里落下来

所以这一阶段最重要的不是再做新的入口，而是把：

**资料库 -> 模拟面试 -> 报告**

这条主链路闭环做完整。

---

## 这次报告场景应该承接什么

这次建议直接承接现有持久化数据，不重新造一套假数据。

优先使用：

- `PersistedInterviewSession`
- `PersistedReportSummary`
- 当前 workbench context

报告场景至少应该能展示：

### 1. 当前报告对应的是哪一轮训练

- 当前 session id
- 主题
- 模式
- 来源资料
- 关联文档

### 2. 这轮训练完成了什么

- 已回答题数
- 总题数
- 当前完成度
- 开始时间
- 结束时间
- 是否完成 / 中断

### 3. 当前最值得关注的弱项

- 弱项标签列表
- 弱项数量
- 第一优先级弱项

### 4. 第一版规则型复盘建议

不急着做 AI 全文报告，第一版先用规则生成即可，例如：

- 如果答题数偏少，提示先补完整轮次
- 如果弱项集中，提示优先专项补练
- 如果训练已完成，提示去看历史记录或继续下一轮

---

## 宇宙页里的报告场景应该长成什么

这次不建议把旧 `report.vue` 原封不动塞进来。

更合适的做法是：

**沿用旧报告页的真实数据结构，但重新组织成宇宙页场景化布局。**

建议报告场景先拆成 4 个区块。

### 1. 报告头部摘要区

负责快速讲清楚：

- 当前报告主题
- 当前来源资料
- 当前训练模式
- 当前完成状态

这一区要像“本轮训练结果总卡”。

### 2. 训练结果概览区

负责展示：

- 已回答 / 总题数
- 完成百分比
- 弱项标签数
- 是否已生成报告摘要

这一区更偏概览和指标。

### 3. 弱项与建议区

负责展示：

- 当前最主要的弱项标签
- 为什么它值得优先补
- 下一步建议动作

这里要让用户知道“看完报告之后该干什么”。

### 4. 行动闭环区

建议保留明确动作：

- 返回资料库
- 继续模拟面试
- 查看历史记录

报告场景不是终点页，而应该是下一步行动的中转页。

---

## 这次和旧 `report.vue` 的关系

旧 `report.vue` 已经有一份“真实报告页接入”的历史开发思路。

这次不重复写一套完全独立逻辑，而是建议：

- 复用旧报告页的数据读取顺序
- 复用现有 session / summary 查找逻辑
- 把展示层改造成宇宙页风格

简单说：

**业务数据层尽量复用，UI 呈现层重新组织。**

---

## 建议的数据读取顺序

宇宙页里的报告场景建议按这个顺序取数据：

1. 优先取当前进行中的或刚完成的 session
2. 如果有对应 summary，优先展示该 summary
3. 如果没有 summary，就用 session 本身做第一版规则型报告
4. 如果既没有当前 session 也没有 summary，则退回最近一条已完成记录

这样可以保证：

- 从 mock 结束进入报告时，有内容
- 从总览点“查看报告”时，也能稳定落到最近可看的结果

---

## 这次不优先做什么

为了避免范围失控，这次先不做：

- AI 长文复盘
- 雷达图 / 趋势图 / 复杂图表
- PDF 导出
- 分享报告
- 多轮报告对比
- 详细题目级逐题分析

这些都可以后置。

本轮最关键的是：

**先让宇宙页报告场景变成真实报告场景。**

---

## 开发顺序建议

建议按下面顺序推进：

### 第 1 步：梳理当前报告场景占位结构

确认宇宙页里 `report` 场景现在有哪些假数据、哪些占位块要替换。

### 第 2 步：抽出报告场景需要的真实数据

从现有 persistence / report 页逻辑里，整理出：

- 当前报告对象
- 当前 session
- 当前 summary
- 基础统计字段

### 第 3 步：先接“真实摘要版报告场景”

第一轮只要求：

- 能看到真实主题
- 能看到真实来源
- 能看到真实完成度
- 能看到真实弱项
- 能看到规则型建议

### 第 4 步：补行动闭环

保证报告场景里的按钮能稳定跳：

- 继续模拟面试
- 返回资料库
- 查看历史

---

## 完成后的目标状态

做完这一轮之后，宇宙页下方的报告场景应该具备这些能力：

- 不是占位说明块
- 能展示真实训练结果
- 能承接刚完成的一轮训练
- 能从总览 / 资料库 / mock 稳定进入
- 能引导用户去做下一步动作

---

## 一句话结论

前面几轮解决的是“怎么开始训练”和“训练依赖什么资料”，这一轮要解决的是：

**训练完成之后，结果如何在宇宙页里真正落地成一份可看的报告。**
