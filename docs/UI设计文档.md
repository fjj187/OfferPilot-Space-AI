# PrepChain UI 设计文档

> 版本：v1.0  
> 日期：2026-05-09  
> 项目定位：基于个人面试资料库的 AI 模拟面试与复盘平台  
> 当前适配页面：工作台首页 / 训练入口页  
> 技术栈：Vue 3 + TypeScript + Naive UI + UnoCSS

---

## 一、设计目标

本项目当前 UI 方案基于已经确认的亮色工作台布局，目标不是做营销型 AI 首页，而是做一个更偏实际使用场景的训练平台。

设计目标如下：

1. 保持工作台气质，强调“资料库 + 训练 + 复盘”的信息结构
2. 整体采用亮色、冷静、克制的视觉风格，避免暗色和炫技式装饰
3. 中间主区域作为内容主舞台，左右区域作为辅助承载
4. 让用户一眼知道这个系统能做什么、当前该做什么、最近的训练状态如何
5. 为后续接入真实功能保留组件扩展空间

---

## 二、整体布局

当前页面采用三栏工作台布局：

### 2.1 左侧导航栏

作用：

- 展示品牌与产品定位
- 承载一级导航
- 放置主行动按钮
- 展示当前训练主题摘要

结构：

1. Logo + 产品名 `PrepChain`
2. 主按钮：`开始训练`
3. 一级导航菜单
4. 底部主题摘要卡片

布局特征：

- 固定宽度，作为稳定导航骨架
- 内容纵向排列
- 视觉上弱于中间主区域，但强于传统后台侧边栏

### 2.2 中间主工作区

作用：

- 作为页面信息主舞台
- 承载搜索、训练计划、资料库概览、训练流程说明
- 后续承接模拟面试核心内容

结构：

1. 顶部搜索区
2. Hero 区域
3. 训练状态卡片
4. 快捷主题标签
5. 最近导入资料区
6. 训练流程区

布局原则：

- 页面视觉重心必须落在中间
- 中间区域信息密度最高
- 所有主要业务动作从中间发起

### 2.3 右侧 AI 引导栏

作用：

- 展示 AI 推荐动作
- 展示专项训练队列
- 展示复盘摘要

结构：

1. AI 引导卡片组
2. 薄弱点训练队列
3. 最近一次复盘快照

布局原则：

- 宽度小于中间主区域
- 只做辅助，不抢主舞台
- 卡片风格要轻量，避免压迫感

## 三、视觉风格

### 3.1 风格关键词

- Light workspace
- Calm productivity
- AI assistant dashboard
- Clean and soft
- Professional but not cold

### 3.2 视觉基调

整体采用浅色、低噪音的工作台风格：

- 大背景使用浅冷色渐变
- 面板统一使用高亮白底
- 边框轻，阴影柔和
- 用蓝紫色作为品牌与重点强调色

避免以下风格：

- 大面积纯黑暗色背景
- 过重的霓虹感和赛博感
- 过度营销化 hero
- 装饰性过强的卡片嵌套

---

## 四、色彩规范

### 4.1 主色体系

建议以当前页面视觉为基准，统一为以下色彩方向：

```css
--color-bg-start: #fbfcff;
--color-bg-end: #f4f7fb;
--color-surface: rgba(255, 255, 255, 0.9);
--color-surface-soft: #f9fbff;
--color-border: #e7ecf5;
--color-border-soft: #edf1f7;

--color-text-primary: #1f2746;
--color-text-secondary: #6d7a92;
--color-text-tertiary: #8d98b0;

--color-brand: #5f79ff;
--color-brand-hover: #556fff;
--color-brand-soft: #eef2ff;
--color-brand-purple: #6c7df7;

--color-success-soft: #eef8f4;
--color-success-text: #2f8f67;
```

### 4.2 使用规则

- 页面背景：浅冷色渐变
- 卡片背景：高透明白或纯白
- 描边：浅灰蓝色细边框
- 主按钮：蓝紫品牌色
- 激活状态：浅蓝紫底 + 深蓝紫文字
- 弱提示：灰蓝文字，不使用高饱和颜色

## 五、圆角、阴影与边框

### 5.1 圆角体系

```css
--radius-pill: 999px;
--radius-card-sm: 14px;
--radius-card-md: 18px;
--radius-card-lg: 24px;
--radius-hero: 28px;
```

使用原则：

- 按钮、标签：圆角胶囊
- 普通信息卡：18px ~ 24px
- Hero 主卡片：28px

### 5.2 阴影体系

```css
--shadow-soft: 0 10px 24px rgba(31, 39, 70, 0.08);
--shadow-card: 0 16px 40px rgba(36, 53, 87, 0.07);
--shadow-brand: 0 18px 30px rgba(95, 121, 255, 0.24);
```

原则：

- 仅使用柔和阴影
- 不做重阴影堆叠
- 不做玻璃态炫光效果

### 5.3 边框体系

- 默认边框：`1px solid #e7ecf5`
- 弱边框：`1px solid #edf1f7`
- 激活边框：使用浅蓝紫色边框

## 六、字体与信息层级

### 6.1 字体建议

- 中文：系统无衬线字体
- 英文：可保持默认无衬线
- 风格目标：清晰、稳定、偏产品化

### 6.2 层级建议

| 用途 | 字号 | 字重 | 说明 |
|------|------|------|------|
| 页面主标题 | 34px | 700 | Hero 主文案 |
| 大区标题 | 22px | 700 | 版块标题 |
| 卡片标题 | 15px~18px | 700 | 模块标题 |
| 正文 | 14px~15px | 400/500 | 普通说明 |
| 辅助文字 | 12px~13px | 400/500 | 描述、meta |
| 数据数值 | 30px | 700/800 | 统计数字 |

原则：

- 标题深色高对比
- 正文灰蓝色
- 说明文字低一级，不抢视线

## 七、页面模块说明

### 7.1 左侧导航模块

#### 品牌区

- 上方 Logo 方块 + 项目名
- Logo 允许使用蓝紫渐变
- 项目副标题使用较浅色说明

#### 主行动按钮

- 文案：`开始训练`
- 使用品牌主色
- 必须是侧栏视觉最强元素

#### 导航菜单

包含：

- 总览
- 资料库
- 模拟面试
- 专项刷题
- 复盘报告
- 训练历史

激活态规则：

- 白底
- 轻阴影
- 深色文字
- 如有 badge，badge 使用浅蓝紫色底

#### 底部训练摘要卡

用于显示：

- 当前训练主题
- 当前岗位方向
- 当前训练重点

此卡片是摘要信息，不承载复杂交互。

### 7.2 顶部搜索栏

作用：

- 搜索资料
- 搜索知识点
- 搜索历史训练

样式规则：

- 胶囊形输入框
- 白底、浅边框
- 左侧图标，右侧不放多余按钮
- 高度要克制，避免占用过多纵向空间

### 7.3 Hero 模块

用于表达平台核心价值。

内容包含：

- 本周训练计划标签
- 一句清晰的价值主张
- 一段补充说明
- 两个核心按钮

按钮规则：

- 主按钮：开始导入资料并训练
- 次按钮：查看样例或报告

此区域应保持当前版本的“信息型 Hero”，不做营销化铺陈。

### 7.4 数据状态卡

当前展示：

- 本周已训练次数
- 资料库文档数

后续可扩展：

- 平均得分
- 薄弱点数量
- 连续训练天数

设计规则：

- 数字大
- 辅助文字少
- 卡片高度一致

### 7.5 主题标签区

当前标签：

- Vue 3
- TypeScript
- 工程化
- 浏览器
- 性能优化
- 场景题

用途：

- 快速切换训练主题
- 快速筛选资料

样式规则：

- 胶囊标签
- 默认白底
- 激活态浅蓝紫底

### 7.6 最近导入资料区

每张资料卡包含：

- 顶部渐变色横条
- meta 信息
- 资料名称
- 摘要说明
- 底部用途说明

设计规则：

- 卡片强调“文档资产”感
- 顶部横条用于区分内容类型
- 信息简洁，不做过重图像化

### 7.7 训练流程区

用于说明平台闭环：

1. 导入资料
2. AI 生成题纲
3. 模拟提问
4. 复盘输出

规则：

- 每一步单独卡片承载
- 图标简洁统一
- 用于建立用户对系统流程的认知

### 7.8 右侧 AI 引导区

#### AI 引导卡片

用于提示用户下一步可以做什么：

- 开始一轮模拟面试
- 针对薄弱点专项刷题
- 生成本周复盘报告

设计规则：

- 首卡允许略高强调
- 其他卡片保持一致风格
- 避免过度花哨

#### 薄弱点队列

用于展示当前知识短板及进度：

- 标题
- 简短说明
- 进度条

规则：

- 每项一张轻卡片
- 进度条颜色用品牌主色

#### 复盘快照

展示：

- 综合得分
- 最近提升趋势
- 关键问题摘要

规则：

- 信息结构必须清晰
- 不做复杂图表，先以摘要为主

## 八、交互规范

### 8.1 按钮

- 主按钮使用品牌色
- 次按钮使用 ghost / outline 风格
- hover 状态只做轻微颜色变化，不做大幅位移动画

### 8.2 卡片

- 整卡可点击时，hover 增加轻阴影
- 不使用明显位移跳动

### 8.3 输入框

- 获焦时边框轻微高亮
- 不使用夸张发光效果

### 8.4 滚动条

- 推荐统一使用浅灰色细滚动条
- 滚动条不应破坏页面的整洁感

---

## 九、响应式原则

### 桌面端

- 三栏布局完整保留
- 中间主区域优先保证宽度

### 中等屏幕

- 右侧栏可以缩窄
- 资料卡从三列降到两列
- 流程卡从四列降到两列

### 小屏设备

- 顶部区域纵向堆叠
- 主内容单列化
- 右侧辅助栏移到下方

原则：

- 不强求桌面结构硬塞到移动端
- 中间主内容优先级最高

---

## 十、当前页面结论

当前确认的页面样式方向是：

- 亮色 AI 工作台
- 左中右三栏布局
- 中间为主，右侧为辅助，左侧为稳定导航
- 蓝紫色作为品牌强调色
- 高透明白卡片 + 浅边框 + 柔和阴影
- 偏产品工作台，不偏营销首页

该方向后续可直接扩展到以下页面：

1. 资料库页
2. 模拟面试页
3. 专项刷题页
4. 复盘报告页
5. 训练历史页

## 十二、文档使用说明

这份文档用于约束 PrepChain 当前版本的页面设计方向。

在后续开发中：

- 新页面优先复用本规范的色彩、间距、圆角和卡片逻辑
- 不要随意引入新的视觉体系
- 所有新增模块都应保持“亮色工作台 + 中间主舞台”的核心原则

如果后续页面大改，可在此文档基础上继续迭代版本号。
