# 资料页参数承接与交互深化开发文档

## 1. 文档目标

这份文档用于规划资料页下一阶段的开发。

当前总览页已经可以把参数带到资料页，但资料页现在只是接住了最基础的筛选逻辑。下一步要做的，不是再堆静态内容，而是把“总览页传过来的上下文”真正落实到资料页的交互里。

这份文档主要回答四个问题：

- 总览页现在会给资料页传什么参数
- 资料页目前已经接住了什么
- 接下来资料页应该继续接住什么
- 真正开发时会修改哪些文件

---

## 3. 总览页现在会传给资料页什么参数

当前总览页跳资料页时，已经会通过 query 传这些字段：

- `topic`
- `docType`
- `source`

这些参数当前主要来自：

- Hero 主按钮
- “查看全部”
- 资料卡点击

### 3.1 参数含义

#### `topic`

示例：

- `vue3`
- `typescript`
- `performance`

作用：

- 告诉资料页当前是从哪个主题上下文进入的

#### `docType`

示例：

- `md`
- `docx`

作用：

- 告诉资料页当前更关注哪类文档

#### `source`

示例：

- `hero-import`
- `library-frontend-notes`
- `library-project-review`

作用：

- 告诉资料页这次跳转来自哪个入口
- 后面可以用来高亮某类文档或某个分组

## 4. 资料页目前已经接住了什么

当前资料页在：

- `src/views/workbench/library.vue`

里已经做了第一层承接。

### 4.1 已实现的内容

已实现：

- 读取 `route.query.topic`
- 读取 `route.query.docType`
- 根据参数初始化 `activeFilter`
- 当 query 变化时重新更新筛选状态

也就是说，目前资料页已经不是完全无感知跳转了，而是能做到：

- 从总览页点资料卡过来后，先切到对应筛选

### 4.2 当前还不够的地方

虽然已经能切筛选，但承接深度还不够。

现在还没有做到：

- 根据 `source` 高亮某个具体文档
- 根据 `topic` 自动选中最相关的文档
- 右侧预览区根据跳转来源做联动
- Hero 入口和资料页导入动作之间形成闭环

所以这一步还只能算“接住了入口”，还没算“真正承接住业务意图”。

---

## 5. 资料页下一步应该深化什么

下一步建议把资料页的深化开发分成四个阶段。

---

## 6. 第一阶段：把 query 承接从“筛选级”升级到“文档级”

### 6.1 目标

当前只做到“切换筛选标签”，下一步要做到：

- 不只是切到某个 filter
- 还要自动定位到更合适的文档

例如：

- 从总览页点“前端八股总纲.md”进来，资料页应该尽量选中对应文档
- 从总览页点某个主题进来，资料页应该优先展示该主题下的第一个相关文档

### 6.2 建议修改文件

- `src/views/workbench/library.vue`
- `src/views/workbench/library.data.ts`

### 6.3 建议补充的数据字段

建议在 `LibraryDocument` 中增加：

- `topicKeys: string[]`
- `sourceKey?: string`

对应文件：

- `src/views/workbench/library.data.ts`

这样做的好处是：

- 页面层不需要靠模糊字符串去猜文档属于哪个主题
- 后面从总览页跳过来时可以更准确匹配

### 6.4 这一阶段开发结果

开发完成后，资料页应该做到：

- 先根据 `topic` 切筛选
- 再根据 `source` 或主题匹配自动选中文档
- 右侧预览区跟着切换到更合理的目标文档

---

## 7. 第二阶段：把资料页做成“导入入口 + 资料整理入口”

### 7.1 目标

让资料页不只是文档列表，而是整个资料流转的工作台入口。

建议实现：

- 从 Hero 进来时，页面能明确告诉用户“下一步去导入资料”
- 点击导入文件 / 文件夹后，列表和预览区马上联动
- 新导入文档自动进入当前列表顶部
- 新导入文档自动成为当前选中项

### 7.2 目前已经有的基础

当前 `library.vue` 已经有：

- `handlePickFiles`
- `handlePickFolder`
- `appendFiles`
- `handleFileChange`

这说明导入能力的雏形已经有了。

### 7.3 下一步建议增强什么

建议增强：

- 当 `source=hero-import` 时，在页面头部或导入卡片处显示当前模式提示
- 导入成功后给出轻反馈
- 导入后自动滚动到新文档
- 根据文件名、标签、类型自动归类

### 7.4 建议修改文件

- `src/views/workbench/library.vue`
- `src/components/library/ImportActionCard.vue`
- `src/components/library/LibraryHeader.vue`
- `src/views/workbench/library.data.ts`

---

## 8. 第三阶段：深化列表区与预览区的联动

### 8.1 目标

资料页当前已经有左右联动的基础，但还可以更像真正的资料工作台。

建议实现：

- 点击左侧文档，右侧预览区即时切换
- 右侧预览区显示更明确的主题信息
- 在列表区对“当前文档为什么被推荐”做轻提示

### 8.2 建议补什么信息

建议在列表项或预览项上加入：

- 主题归属
- 来源标签
- 推荐原因
- 是否已解析

### 8.3 建议修改文件

- `src/components/library/DocumentListItem.vue`
- `src/components/library/DocumentPreviewPanel.vue`
- `src/views/workbench/library.vue`
- `src/views/workbench/library.data.ts`

### 8.4 这一阶段开发结果

完成后资料页会更像：

- 左边是可筛选、可切换的资料索引
- 右边是当前资料的解析与预览面板

而不是只有“一个列表 + 一个说明区”

---

## 9. 第四阶段：为后续 AI 能力预留结构

### 9.1 目标

当前先不接接口，但资料页的数据结构要为后面这些能力留口：

- 文档切片
- 文档摘要
- 文档知识点提取
- 文档推荐训练方向

### 9.2 建议补充的数据结构

建议未来在 `LibraryDocument` 里扩展：

- `topicKeys`
- `knowledgePointCount`
- `sliceCount`
- `recommendedUsage`
- `analysisStatus`

建议修改文件：

- `src/views/workbench/library.data.ts`
- 未来可能新增 `src/types/library.ts`

### 9.3 当前先不要急着做什么

当前先不要急着做：

- 真实 Word 解析
- 真正的 Markdown 结构化切片
- 向量检索
- 后端上传接口

因为这一步最重要的是把前端交互路径先做扎实。

## 10. 真正开始开发时推荐的文件修改顺序

建议按下面顺序来改。

### 第一步：先整理数据结构

修改：

- `src/views/workbench/library.data.ts`

先做：

- 给文档补主题键
- 给文档补来源键
- 让筛选、定位和推荐不靠硬编码字符串

### 第二步：再改页面层承接逻辑

修改：

- `src/views/workbench/library.vue`

先做：

- query -> filter
- query -> selectedDocumentId
- source -> 默认定位逻辑

### 第三步：再改展示组件

修改：

- `src/components/library/DocumentListItem.vue`
- `src/components/library/DocumentPreviewPanel.vue`
- `src/components/library/ImportActionCard.vue`
- `src/components/library/LibraryHeader.vue`

再做：

- 当前文档高亮强化
- 来源提示
- 导入提示
- 上下文提示

## 11. 当前阶段最重要的开发原则

一句话总结：

不要只让资料页“看到参数”，要让资料页“理解参数并作出正确联动”。

也就是说，下一步资料页的重点不是新增更多卡片，而是把这条链路做实：

1. 总览页传来参数
2. 资料页识别参数含义
3. 自动切筛选
4. 自动选中合理文档
5. 自动更新右侧预览
6. 让用户感受到页面真的接住了上下文

---

## 12. 下一步做完之后，资料页应该达到什么效果

理想状态下，下一步做完后用户会感受到：

- 从总览页点不同资料入口过来，资料页反应不一样
- 不是只有标签变了，而是文档列表和右侧预览都跟着变
- 点击 Hero 的“导入资料并开始”后，资料页像真正的资料工作台入口
- 新导入文档能自然进入当前上下文流程

---

## 13. 后续在整个项目中的位置

资料页是后面这几条能力链的基础：

- 总览页 -> 资料页 -> 模拟面试页
- 总览页 -> 资料页 -> 专项刷题页
- 总览页 -> 资料页 -> 复盘报告页

所以资料页这一步一旦做稳，后面接：

- mock interview
- practice
- report

都会顺很多。
