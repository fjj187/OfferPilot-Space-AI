# 使用者阅读文档

这是一份面向项目使用者的快速说明，帮助你在本地启动并使用 `chatgpt-vue3-light-mvp`。

## 1. 项目简介

`chatgpt-vue3-light-mvp` 是一个基于 `Vue 3 + TypeScript + Vite + Naive UI` 的 AI 对话前端项目。

它的特点是：

- 支持流式输出，能看到逐字生成效果
- 支持 Markdown 渲染
- 支持代码高亮
- 支持 Mermaid 图表与 KaTeX 数学公式
- 支持切换不同模型
- 支持本地模拟模式，便于前端开发联调

当前项目更偏向一个可二次开发的前端 MVP，对话模式默认是单轮问答。

## 2. 运行环境

请先确认本机具备以下环境：

- `Node.js >= 22.12`
- `npm >= 10` 或 `pnpm >= 10`

如果你只是本地体验项目，使用 `npm` 即可。

## 3. 安装依赖

在项目根目录执行：

```bash
npm install
```

如果你本机已安装 `pnpm`，也可以使用：

```bash
pnpm install
```

## 4. 启动项目

在项目根目录执行：

```bash
npm run dev
```

启动成功后，在浏览器打开本地地址。

当前这份项目在本机环境中实际启动地址通常是：

```text
http://localhost:2048/
```

如果端口被占用，Vite 可能会自动切换到别的端口，请以终端输出为准。

## 5. 初次使用建议

第一次打开项目时，建议先按下面的顺序体验：

1. 打开首页，确认页面能正常加载
2. 先使用默认模拟模型进行一次提问
3. 观察回复是否为流式输出
4. 测试 Markdown、代码块、列表等内容渲染效果
5. 再决定是否切换到真实模型

这样可以先验证前端功能是否正常，再处理模型接口配置问题。

## 6. 模拟模式与真实模型

项目支持两种使用方式：

### 6.1 模拟模式

模拟模式适合：

- 前端开发联调
- 演示页面效果
- 暂时没有 API Key 的情况

这种模式下，不依赖真实大模型接口，也能看到基础对话展示效果。

### 6.2 真实模型模式

如果你希望接入真实大模型，需要配置对应平台的 API Key。

当前项目已预留这些环境变量：

```env
VITE_SPARK_KEY=key123456:secret123456
VITE_SILICONFLOW_KEY=sk-xxxxxx
VITE_MOONSHOT_KEY=sk-xxxxxx
VITE_DEEPSEEK_KEY=sk-xxxxxx
```

## 7. 配置环境变量

在项目根目录新建 `.env` 文件。最简单的方式是参考 `.env.template` 内容手动创建。

示例：

```env
VITE_SPARK_KEY=你的SparkKey:你的SparkSecret
VITE_SILICONFLOW_KEY=你的SiliconFlowKey
VITE_MOONSHOT_KEY=你的MoonshotKey
VITE_DEEPSEEK_KEY=你的DeepSeekKey
```

说明：

- `Spark` 需要用 `APIKey:APISecret` 的格式
- 其他大多数平台通常是 `sk-xxxxxx` 形式
- 没有使用的模型可以先留空

配置完成后，重启开发服务使配置生效。

## 10. 常见问题

### 10.1 项目能打开，但无法正常回答

可能原因：

- 当前使用的是需要 API Key 的模型，但 `.env` 没配
- API Key 配错了
- 接口服务本身不可用
- 本地网络无法访问对应模型平台

建议先切回模拟模式，确认前端流程没有问题。

### 10.2 启动失败

请依次检查：

- Node 版本是否过低
- 依赖是否安装完成
- 终端中是否有端口占用或权限错误

可以重新执行：

```bash
npm install
npm run dev
```

### 10.3 页面空白或样式异常

可能是：

- 依赖未完整安装
- 浏览器缓存导致资源异常
- 启动日志中存在编译错误

建议刷新页面，必要时重启项目。
