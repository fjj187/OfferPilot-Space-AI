# 工作台容器高度与滚动注意事项

## 1. 这份文档是干什么的

这份文档专门说明当前工作台页面的：

- 容器高度是怎么一层层传下来的
- 为什么资料页、总览页会出现内部滚动条
- 哪一层该滚，哪一层不该滚
- 后面继续开发页面时，哪些写法容易把布局弄乱

适用范围：

- `src/views/workbench/layout.vue`
- `src/components/workbench/WorkbenchContentShell.vue`
- `src/views/workbench/overview.vue`
- `src/views/workbench/library.vue`
- `src/views/workbench/mock-interview.vue`
- `src/views/workbench/practice.vue`
- `src/views/workbench/report.vue`
- `src/views/workbench/history.vue`

---

## 2. 当前布局结构

现在工作台页面的高度与滚动关系，大致是下面这条链路：

1. `layout.vue` 中的 `.workspace`
2. `WorkbenchContentShell.vue` 中的 `.content-page`
3. `WorkbenchContentShell.vue` 中的 `.content-shell`
4. `WorkbenchContentShell.vue` 中的 `.content-main / .content-aside`

可以把它理解成：

- `.workspace` 提供整个右侧工作区的可用高度
- `.content-page` 接住表头下面剩余空间
- `.content-shell` 是固定高度的白色内容容器
- `.content-main / .content-aside` 在白色容器内部各自滚动

---

## 3. 每一层分别负责什么

### 3.1 `layout.vue` 的 `.workspace`

文件：

- `src/views/workbench/layout.vue`

当前核心样式：

```scss
.workspace {
  height: 100%;
  display: flex;
  flex-direction: column;
}
```

它的职责是：

- 让整个右侧工作台区域撑满父容器高度
- 让头部 `WorkbenchHeader` 在上面
- 让下面的页面内容区占据剩余空间

如果这一层没有 `height: 100%` 或者不是纵向 `flex`，下面的内容容器就拿不到稳定高度。

---

### 3.2 `WorkbenchContentShell.vue` 的 `.content-page`

文件：

- `src/components/workbench/WorkbenchContentShell.vue`

当前核心样式：

```scss
.content-page {
  flex: 1;
  min-height: 0;
  padding: 0 30px 8px;
}
```

它的职责是：

- 占满 `workspace` 里表头以下的剩余高度
- 允许自己收缩，不把父容器撑爆

这里最关键的是：

- `flex: 1`
- `min-height: 0`

其中 `min-height: 0` 很重要。  
如果少了它，很多浏览器会默认让 flex 子项按内容高度撑开，结果就是：

- 白色容器被内容越撑越高
- 内部滚动条消失
- 页面看起来像“没有滚动”

---

### 3.3 `WorkbenchContentShell.vue` 的 `.content-shell`

当前核心样式：

```scss
.content-shell {
  box-sizing: border-box;
  height: 100%;
  min-height: 0;
  overflow: hidden;
}
```

它的职责是：

- 作为白色大容器
- 高度跟随 `.content-page`
- 自己不负责滚动
- 把溢出裁掉，交给内部区域滚动

这里最关键的是：

- `height: 100%`
- `overflow: hidden`

这意味着：

- 白色容器本身是定高的
- 内容多了以后，不是让它自己无限撑高
- 而是让内部主区和右侧区各自出现滚动条

---

### 3.4 `WorkbenchContentShell.vue` 的 `.content-main / .content-aside`

当前核心样式：

```scss
.content-main,
.content-aside {
  min-height: 0;
  overflow-y: auto;
  padding-bottom: 12px;
}
```

它们的职责是：

- 真正承接内容滚动
- 主区滚主区
- 右栏滚右栏

这也是为什么现在：

- 总览页可以滚动
- 资料页可以滚动
- 右侧卡片多了也不会把整个外层盒子撑开

---

## 4. 两栏布局是怎么工作的

当页面需要右侧栏时，会给 `WorkbenchContentShell` 传：

```vue
<WorkbenchContentShell has-aside aside-width="minmax(240px, 315px)">
```

然后通过内联样式给白色容器设置：

```ts
gridTemplateColumns: `minmax(0, 1.7fr) ${asideWidth}`
```

也就是：

- 左边主内容更宽
- 右边侧栏固定在一个合理范围内

当前这样写的原因是为了避免之前那种：

- 使用 CSS 变量拼接列宽
- 某些情况下列宽表达式失效
- 两栏退化成单列
- 右侧卡片被挤到下面

---

## 5. 为什么之前会出现“资料页没滚动条”

之前的现象不是最终设计效果，而是容器关系没有完全收住。

常见成因有这几类：

1. 父层没有稳定高度
2. flex 子项缺少 `min-height: 0`
3. 外层容器在撑高，内层没有接管滚动
4. 某个页面没有走统一容器，和别的页面逻辑不一致

表现出来就是：

- 内容看起来直接往下铺开
- 底部卡片会被下面区域挡住
- 没有出现应该出现的滚动条

现在修好后，滚动条出现在内容区本身，这才是正确状态。

---

## 6. 后续开发时最容易踩的坑

### 6.1 不要随便删 `min-height: 0`

尤其是下面这些层：

- `.content-page`
- `.content-main`
- `.content-aside`

很多时候页面“不能滚”或者“容器被撑爆”，根因就是这里被删了。

---

### 6.2 不要让 `.content-shell` 自己滚

当前白色容器是：

```scss
overflow: hidden;
```

如果你把它改成：

```scss
overflow-y: auto;
```

会带来几个问题：

- 主区和右栏失去独立滚动
- 整个白色容器一起滚
- 左右内容区的视觉对齐会变差

---

### 6.3 不要让页面内容直接把外层撑高

比如某些页面里如果加了：

- 超大固定高度块
- 没有限制高度的绝对定位内容
- 不受控的长列表容器

就可能导致内部滚动区域失效。

开发时要优先检查：

- 是不是某个子元素没有 `min-width: 0 / min-height: 0`
- 是不是某个区域被写成了 `height: auto` 然后无限撑开

---

### 6.4 有右栏的页面要统一走 `WorkbenchContentShell`

当前这些页面已经接入统一容器：

- `overview.vue`
- `library.vue`
- `mock-interview.vue`
- `practice.vue`
- `report.vue`
- `history.vue`

如果后面又新建页面，不要再单独手写另一套：

- `workspace-body`
- `main-stage`
- `right-rail`

否则很容易再次出现：

- 某些页间距变了
- 某些页没滚动条
- 某些页右栏掉到下面

---

## 7. 开发新页面时的推荐写法

### 7.1 单栏页面

```vue
<WorkbenchContentShell>
  <!-- 页面内容 -->
</WorkbenchContentShell>
```

适合：

- 模拟面试页
- 专项刷题页
- 报告详情页

---

### 7.2 两栏页面

```vue
<WorkbenchContentShell has-aside aside-width="minmax(240px, 315px)">
  <div>主内容</div>

  <template #aside>
    <div>右侧栏内容</div>
  </template>
</WorkbenchContentShell>
```

适合：

- 总览页
- 资料页
- 以后带 AI 引导侧栏的页面

---

## 8. 改完页面后要检查什么

每次改完工作台页面，建议至少检查下面几点：

1. 页面是否还能滚动
2. 右侧栏是否仍然在右边，没有掉到底部
3. 白色容器底部是否还有合理留白
4. 内容底部是否会被下方区域遮住
5. 资料页、总览页、报告页的滚动行为是否一致
6. 窄屏时两栏是否会按媒体查询正常变成单栏

---

## 9. 这份结构的核心原则

一句话总结：

不是让“整个页面无限往下长”，而是让“固定高度容器内部滚动”。

再具体一点，就是：

- 父层负责给高度
- 白色容器负责收口
- 主区和侧栏负责滚动

只要后面继续开发时不破坏这三层职责，这套布局就会一直稳定。
