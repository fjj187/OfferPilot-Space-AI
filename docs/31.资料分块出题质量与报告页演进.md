> 状态：分块 / 出题质量 **已实现（规则层）**；报告页「题目 + 答案 + 解析」**仅类型预留，UI 未做**

> 单测：`__tests__/material-question-pipeline.test.ts`

## 1. 背景与目标

### 1.1 业务问题

资料刷题主链路（资料页 → 生成题库 → 模拟面试 → 报告）在 MVP 阶段已打通，但存在两类缺口：

1. **出题覆盖不足**：常见面试 md 结构为 `### 题面？` + 列表式参考答案，旧逻辑因「正文像答案」而跳过出题，导致 `isShortfall`（题量不足）频繁出现。
2. **报告信息偏薄**：`PersistedReportSummary.answerSnapshot`（作答摘录）多为字符串数组，缺少「每题题目 / 你的答案 / 资料解析」结构化复盘，与用户对报告页的期望不一致。

### 1.2 本文档范围

| 范围 | 状态 |
|------|------|
| 分块规则优化 | ✅ 已落地 |
| 出题规则优化 + `referenceAnswer` 透传 | ✅ 已落地 |
| 题组 `shortfall` 行为说明 | ✅ 本文定义 |
| 报告页 UI 与 `buildReportSummary` 填充 `questionReviews` | ⏳ 下一阶段 |

### 1.3 不在本文实现范围

- LLM 自动出题 / 改写资料
- docx 资料解析策略
- 旧 workbench 报告页改造（宇宙页 `SpaceReportScene` 优先）

## 3. 分块规则（当前实现）

文件：`src/services/material/material-chunk-splitter.ts`

### 3.1 标题识别

- 正则：`^(#{1,4})\s+(.+)$`，支持 **H1～H4**（此前仅 H1～H3）。
- `level` = `#` 个数；`heading` = 标题文本（不含 `#`）。

### 3.2 段落合并与丢弃

- 遇到新标题时 `flushSection`，将上一标题下正文合并为一个 chunk。
- 正文经 `normalizeSectionText`：去掉 `---` / `___` 分隔线。
- **最短有效段落**：`minSubstantiveChunkLength = 12`（由 24 下调，避免短答案段被整段丢弃）。
- 若无任何 chunk，整篇作为 `资料正文` 单段兜底。

### 3.3 超长切分

- 单段超过 `maxChunkLength = 1000` 时按窗口切分，标题追加 `（续 N）`。
- 切片需 `>= minChunkLength = 80`，或已到文末。

### 3.4 推荐资料写法（利于出题）

```markdown
## 章节名（可选，不单独出题 unless 满足讲解规则）

### 面试题面？（推荐：三级标题 + 问号）

- 答案要点 1
- 答案要点 2
```

## 4. 出题规则（当前实现）

文件：`src/services/material/material-question-builder.ts`、`src/services/material/material-prompt.ts`

### 4.1 题面型标题 `isQuestionLikeHeading`

满足其一即视为题面（排除 `资料正文`、`段落 N`）：

- 标题以 `？` / `?` 结尾
- 或以常见问句词开头：`什么是`、`如何`、`为什么`、`有哪些`、`区别`、`对比` 等（见 `material-prompt.ts` 正则）

### 4.2 面试题 chunk `isInterviewQuestionChunk`

在题面型标题前提下：

- `chunk.level >= 2`，**或** 标题带问号  

即：**二级标题带问句** 也会出题（不再要求必须三级标题）。

### 4.3 每 chunk 出题优先级（最多 2 题 / chunk）

1. **面试题**（`isInterviewQuestionChunk`）  
   - `title` = 规范化标题  
   - `prompt` = 标题原文（`buildHeadingQuestionPrompt`）  
   - **不因正文像答案而跳过**  
   - `referenceAnswer` = `extractMaterialReferenceAnswer(chunk.text)`（最长 480 字）

2. **章节讲解**（`level <= 2`，正文 ≥ 48 字，且 `!looksLikeAnswerBody`）  
   - `title` = `请讲解：{heading}`  
   - 模板 prompt，带章节名

3. **题面兜底**（题面型标题但未命中 1）  
   - 仍按题面出题

4. **代码题**（正文含 fenced code，且已有题数 < 2）  
   - `questionType = code`

5. **概念兜底**（以上均无，但有有效标题）  
   - 即使 `looksLikeAnswerBody`，只要有标题也会出一题（修复 shortfall 关键）

### 4.4 `looksLikeAnswerBody` 语义（未删除，作用域缩小）

正文更像「答案要点」当：

- 以列表项 / 有序列表开头，或  
- 长度 ≥ 48 且含换行  

**不再**单独作为「禁止出题」条件；仅用于讲解题（分支 2）与 concept prompt 是否附带 `formatMaterialReference`。

### 4.5 题库产出 `MaterialQuestionPool`

| 字段 | 说明 |
|------|------|
| `chunks` | 全部分块 |
| `questions` | 扁平化题目列表 |
| `status` | 无题 → `error`，文案：「当前资料内容过短，暂无法生成练习题」 |

## 5. 组卷与 shortfall

文件：`src/services/material/material-question-group-builder.ts`

### 5.1 默认配置

```ts
defaultMaterialGroupCompileOptions(): { count: 5, orderMode: 'chapter' }
```

### 5.2 选题逻辑

- 先按 `difficultyFilter`（若有）过滤  
- 再按 `orderMode`：`chapter` | `difficulty_ladder` | `single_difficulty` 排序  
- 顺序取前 `count` 道，**每题 id 去重**

### 5.3 shortfall 定义

```ts
isShortfall = actualCount < requestedCount
```

**合法 shortfall 场景**（产品需接受或提示）：

- 资料内独立题面章节数 < 请求题数（例如只有 3 个 `###` 题却要 5 题）
- `difficultyFilter` / `single_difficulty` 过滤后不足

**不应再因「正文是答案列表」导致整库题数过少**（已通过 4.3 修复）。

### 5.4 题组项新增字段

`PersistedPracticeQuestionGroupItem.referenceAnswer`：从 `MaterialQuestionItem.referenceAnswer` 透传，供模拟面试侧栏「资料依据」与**未来报告**使用。

## 6. 类型与报告逐题复盘（已实现）

### 6.1 资料题 `MaterialQuestionItem`

```ts
referenceAnswer?: string  // 资料参考答案摘要
sourceHeading?: string    // 规范化章节标题
```

### 6.2 报告复盘项 `PersistedReportQuestionReviewItem`（workbench.ts）

```ts
{
  questionId: string
  questionTitle: string
  userAnswer: string
  referenceAnswer?: string  // 资料解析
  aiFeedback?: string       // 该题 AI 反馈
}
```

### 6.3 `PersistedReportSummary.questionReviews`

- `buildReportSummary` 会遍历 `visibleQuestionThreads`，按 `threadId` 取用户作答与 AI 反馈，并从题组项读取 `referenceAnswer`。
- `finishMockSession` 持久化摘要时保留本地 `questionReviews`（远程报告 API 无该字段时仍合并写入）。
- `answerSnapshot` 仍保留，供旧版摘要摘录与远程 hydration 兼容。

## 7. 验收标准

### 7.1 自动化

```bash
pnpm exec vitest run __tests__/material-question-pipeline.test.ts
```

覆盖：

1. 典型面试 md（5 个 `###` 题面）→ `questions.length >= 4`，且多数带 `referenceAnswer`
2. 默认 5 题组卷 → `actualCount === 5`，`isShortfall === false`
3. `####` 标题 → 2 个 chunk，`level === 4`

### 7.2 手工（资料刷题）

| 步骤 | 预期 |
|------|------|
| 资料页选含多道 `### …？` 的 md，生成题库 | 状态「已生成 N 题」，N 与题面数接近 |
| 开始刷题，默认 5 题 | 无 shortfall 提示（题面足够时） |
| 模拟面试侧栏 | 资料依据 5 行截断；作答建议非重复题面 |
| 切题 / 下一题 | 顺序递增，不跳回未作答题 |

## 8. 报告页逐题复盘（已实现）

> **目标**：报告页展示「题目 + 你的答案 + 资料解析（+ AI 反馈）」，而不是仅整体弱项分析。

### 8.1 建议 UI 结构（`SpaceReportScene`）

每题一张卡片：

| 区块 | 数据来源 |
|------|----------|
| 题目标题 | `thread.title` / `groupItem.title` |

   - `answerSnapshot` 可保留兼容或改为派生

2. **`useMockInterviewSpaceReportScene.ts`**  
   - 新增 `reportQuestionReviews` computed，供 `SpaceReportScene` / `SpaceReportPreviewModal`

## 10. 风险与已知限制

1. **题量上限**：仍受资料内「独立题面章节数」限制；不能凭空凑满 5 题。  
2. **参考答案质量**：规则抽取，非 LLM 总结；列表原文可能较长，报告侧需再截断展示。  
3. **`looksLikeAnswerBody`**：对短列表仍可能误判为答案体，但不阻止出题。  
4. **历史报告无 `questionReviews`**：可从 `session.messagesSnapshot` 回退出题面/作答/反馈，但可能没有 `referenceAnswer`。
