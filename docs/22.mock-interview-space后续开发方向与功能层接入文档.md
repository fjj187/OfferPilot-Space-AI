# 22.mock-interview-space 后续开发方向与功能层接入文档

## 1. 当前阶段判断

`/showcase/mock-interview-space`（模拟面试星球页）已经完成从视觉展示页到五场景工作台雏形的演进：

1. `overview`（总览场景）已经承接训练进度、当前上下文和入口动作。
2. `library`（资料库场景）已经具备导入入口、资料列表、当前训练资料和文档预览。
3. `mock`（模拟面试场景）已经具备题目区、消息区、回答区和当前引用资料信息。
4. `feedback`（专项刷题场景）已经具备弱点、题型、题量和训练入口。
5. `report`（复盘报告场景）已经具备报告摘要、弱点、建议和历史入口。
6. `SpaceSceneHeader.vue`（宇宙场景头部组件）已经统一场景头部结构。
7. 近期已处理资料库左右布局、右侧 sticky（粘性定位）行为、惯性滚动反馈和文档预览去重问题。

这说明页面已经不再是“先把内容画出来”的阶段，而是进入：

**展示层封板 -> 状态契约收敛 -> 功能层接入** 的过渡阶段。

后续开发不能继续无限打磨视觉细节，也不能立刻把所有真实功能都塞进页面。正确方向是先把场景结构和状态边界固定，再按主链路接入功能层。

---

## 2. 接下来开发方向

下一阶段的核心方向不是新增更多场景，而是让现有五个场景从“看起来像完整产品”变成“能承接真实训练流”的稳定容器。

建议按三条线推进：

### 2.1 第一条线：展示层封板

目标是减少后续功能接入时的 UI（用户界面）返工。

需要完成：

1. 五个场景标题、说明、内容起点高度稳定。
2. 资料库右侧 sticky（粘性定位）和惯性动效不再影响列表滚动。
3. `DocumentPreviewPanel.vue`（文档预览组件）只展示必要信息，不重复展示主题和标签。
4. 移动端和窄屏下所有场景不横向溢出。
5. 场景切换、胶囊下滑、轨道点击、自动轮播没有明显回归。

展示层封板后，视觉类修改只处理阻断功能接入的问题，不再做大范围视觉重排。

### 2.2 第二条线：状态契约收敛

目标是让各场景共享同一套业务上下文，而不是每个场景各自拼数据。

需要明确这些状态：

1. 当前训练资料：来自资料库选中文档。
2. 当前训练主题：来自总览、资料库或用户选择。
3. 当前训练模式：模拟面试、专项刷题、复盘。
4. 当前 session（训练轮次）：承接问题、回答、流式消息和报告。
5. 当前 report summary（报告摘要）：承接复盘场景和历史入口。

建议优先检查并收敛：

1. `useOverviewLaunchState.ts`（总览启动状态组合函数）。
2. `useLibraryWorkspaceState.ts`（资料库工作区状态组合函数）。
3. `useMockInterviewSpaceMockState.ts`（星球页模拟面试状态组合函数）。
4. `useMockInterviewSpaceReportScene.ts`（星球页报告场景组合函数）。
5. `useWorkbenchPersistence.ts`（工作台持久化组合函数）。

状态契约收敛的结果应该是：从资料库选中文档后，进入模拟面试、专项刷题、报告时都能解释“当前训练基于什么资料和主题”。

### 2.3 第三条线：功能层接入

功能层不是指继续加按钮，而是让真实能力开始驱动场景内容。

第一批应该接入这些功能：

1. 资料导入后的真实解析状态和错误状态。
2. 当前资料对模拟面试 prompt（提示词）的影响。
3. 模拟面试真实 session（训练轮次）创建、继续、结束。
4. 回答提交后的流式追问、停止、错误恢复。
5. 训练结束后的报告摘要生成和复盘场景读取。
6. 专项刷题基于报告弱点生成训练题组。

不建议第一批接入：

1. 多模型配置中心。
2. 完整账号体系。
3. 云端数据库同步。
4. 复杂文件夹索引和全文搜索。
5. 大规模题库管理后台。

这些功能会显著扩大边界，应该等主链路闭环稳定后再做。

---

## 3. 什么时候接入功能层

功能层接入不应该按日期判断，而应该按验收门槛判断。

满足以下条件后，就可以正式进入功能层：

1. 五个场景切换稳定，标题区和内容区不再大幅改布局。
2. 资料库场景的当前训练资料、文档预览、导入入口和列表选择行为稳定。
3. 模拟面试场景能稳定展示当前引用资料和当前问题，不需要再改大结构。
4. 报告场景能稳定承接最新训练摘要，不需要再重做布局。
5. `mock-interview-space.vue`（模拟面试星际页面主文件）继续保持装配职责，没有新增大段业务 UI（用户界面）。
6. 本轮改动文件通过 `eslint`（代码规范检查）和 `npm run build`（生产构建验证）。
7. 含中文源码文件通过乱码扫描，避免编码污染继续扩散。

如果以上条件未满足，仍处于展示层封板阶段。

如果以上条件满足，下一张任务单就应该从“视觉收尾”切换为：

**`23.mock-interview-space功能层第一阶段接入开发文档`**

## 4. 功能层第一阶段建议做什么

功能层第一阶段建议只做一条最短闭环：

**资料库选择资料 -> 进入模拟面试 -> 真实流式问答 -> 结束训练 -> 生成报告摘要 -> 报告场景展示。**

这条闭环优先级最高，因为它能证明星球页不是展示壳，而是完整训练产品。

### 4.1 资料库到模拟面试

目标：

1. 用户在资料库选择某份文档。
2. 点击“去模拟面试”。
3. 模拟面试场景明确使用这份文档作为上下文。
4. 问题生成和追问能读取文档摘要、标签、主题和原文片段。

验收：

1. 切换资料后再进入模拟面试，问题上下文随之变化。
2. 页面刷新后仍能恢复当前训练资料。
3. 资料缺失时有明确兜底提示，而不是静默使用假数据。

### 4.2 模拟面试 session 闭环

目标：

1. 创建一轮真实 session（训练轮次）。
2. 用户提交回答后触发真实流式追问。
3. 支持停止、错误提示、继续回答。
4. session summary（训练摘要）写入持久化层。

验收：

1. 一轮训练至少能完成“问题 -> 回答 -> 追问 -> 再回答”。
2. 中途切换场景再回来，当前 session 不丢。
3. 结束训练后能生成可读取的摘要。

### 4.3 报告场景读取真实摘要

目标：

1. 报告场景读取最新 session summary（训练摘要）。
2. 展示弱点、建议、表现概览。
3. 支持从报告返回继续训练。

验收：

1. 报告不是固定 mock（模拟）文案。
2. 报告内容和刚才的训练回答有可解释关系。
3. 历史入口能看到这一轮训练摘要。

---

## 5. 功能层第二阶段再做什么

第一阶段闭环稳定后，再进入第二阶段：

1. 专项刷题接入报告弱点，生成真实训练题组。
2. 资料库导入增强，支持更可靠的解析反馈。
3. 训练历史列表和报告详情增强。
4. 总览场景展示真实连续训练进度。
5. 补充更完整的异常恢复和空状态。

第二阶段的重点是增强“复训能力”，不是继续扩展视觉表现。

---

## 6. 仍然不要做的事

在功能层第一阶段之前，不建议做：

1. 重写星球舞台视觉。
2. 新增第六、第七个轨道场景。
3. 把旧工作台页面大规模搬回星球页。
4. 为了接功能把业务状态写回 `mock-interview-space.vue`（模拟面试星际页面主文件）。
5. 在没有状态契约的情况下直接接多个接口。
6. 为了“看起来更完整”继续堆静态卡片。

这些都会让页面变成另一个难维护的大文件，和文档 18 的边界规范冲突。

---

## 7. 推荐文件落点

功能层接入时，建议按下面落点开发：

1. 页面装配：`src/views/showcase/mock-interview-space.vue`（模拟面试星际页面主文件），只做薄装配。
2. 场景挂载：`SpaceContentPanel.vue`（宇宙内容面板组件）。
3. 资料库 UI：`SpaceLibraryScene.vue`（资料库场景组件）。
4. 模拟面试 UI：`SpaceMockScene.vue`（模拟面试场景组件）。
5. 专项刷题 UI：`SpacePracticeScene.vue`（专项刷题场景组件）。
6. 报告 UI：`SpaceReportScene.vue`（复盘报告场景组件）。
7. 模拟面试状态：`useMockInterviewSpaceMockState.ts`（星球页模拟面试状态组合函数）。
8. 报告拼装：`useMockInterviewSpaceReportScene.ts`（星球页报告场景组合函数）。
9. 持久化：`useWorkbenchPersistence.ts`（工作台持久化组合函数）。
10. 接口和流式能力：优先复用已有 `useInterviewStream`（面试流式对话组合函数）。

如果某个功能需要跨越多个场景，先定义状态契约，再改 UI（用户界面）。不要让场景组件互相读彼此内部状态。

## 9. 一句话结论

接下来的开发方向是：

**先把星球页展示层封板，再收敛跨场景状态契约，随后只接一条最短真实训练闭环。**

什么时候接入功能层：

**当五个场景布局稳定、当前资料和 session 状态契约明确、构建与乱码扫描通过后，就进入功能层第一阶段。**
