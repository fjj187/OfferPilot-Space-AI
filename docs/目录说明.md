# 项目目录说明

> 面向新成员的快速 onboarding。详细演进计划见 [`项目整理方向.md`](./项目整理方向.md)。

## 仓库顶层

```
chatgpt-vue3-light-mvp-main/
├── frontend/            # 前端（Vue 3 + Vite）
│   ├── src/
│   ├── __tests__/
│   ├── public/
│   └── package.json
├── backend/             # 后端（Express + TypeScript）
├── docs/                # 项目文档
├── package.json         # 根脚本（pnpm workspace 入口）
├── pnpm-workspace.yaml
├── pnpm-lock.yaml
└── README.md
```

**前后端分离**：前端在 `frontend/`，后端在 `backend/`，根目录 `pnpm dev` / `pnpm dev:backend` 分别启动。

## 前端 `frontend/src/` 分层

| 目录 | 职责 | 何时放这里 |
|------|------|-----------|
| `views/showcase/` | **主产品**页面入口 | 宇宙模拟面试页及 showcase 类页面 |
| `views/workbench/` | **Legacy** 旧工作台（已冻结） | 不再新增；见目录内 README |
| `components/showcase/mock-interview-space/` | 宇宙页专用 UI | 仅宇宙页使用的场景、面板、动画壳 |
| `components/mock-interview/` 等 | **领域共享** UI | workbench 与 showcase 都可能用的面试/资料/报告组件 |
| `components/Layout/`、`Navigation/` 等 | **通用** UI | 无业务语义的可复用组件 |
| `composables/` | 组合函数（状态与流程） | 按子目录：`showcase/`、`interview/`、`workspace/`、`ui/` |
| `services/` | API、SSE、领域服务 | 有 IO 或业务编排（如 `interview/`、`practice/`、`sse/`） |
| `services/storage/` | 本地持久化（localStorage） | 资料库、题池、会话等键值读写 |
| `utils/` | 纯函数工具 | 格式化、签名、无状态转换 |
| `types/` | 共享 TypeScript 类型 | 跨模块领域类型 |
| `router/` | 路由与守卫 | 入口重定向、legacy 拦截 |
| `config/` | 环境与产品开关 | 如 `product.ts` 控制旧工作台是否暴露 |

## 后端 `backend/src/` 分层

```
controllers/   → HTTP 入参校验与响应
routes/        → 路由注册
services/      → 业务逻辑
providers/     → LLM / 引擎适配
storage/       → 会话与报告持久化
types/         → 后端类型
utils/         → 提示词构建、SSE、环境变量等
```

前后端领域命名尽量对齐：`interview`、`practice-pool`、`report`。

## 组件放置决策（三级）

新增 Vue 组件时按顺序判断：

1. **产品专用** — 只给宇宙页用 → `components/showcase/mock-interview-space/`（或 `scenes/`）
2. **领域共享** — 多个产品面或场景复用 → `components/<domain>/`（如 `mock-interview/`、`report/`）
3. **通用 UI** — 与业务无关 → `components/<技术名>/`（如 `Layout/`、`Pagination/`）

## 默认入口与 Legacy 工作台

- 默认首页：`/` → `/showcase/mock-interview-space`（宇宙模拟面试页）
- 旧工作台路由 `/workspace/*` **仍保留在代码中**，但默认不可访问
- 开关：`src/config/product.ts` → `LEGACY_WORKBENCH_ENABLED`
  - `false`（默认）：访问 `/workspace` 会重定向到宇宙页
  - `true`：仅用于本地对比 legacy 行为

## 常用命令

```bash
pnpm i                          # 仓库根目录安装全部 workspace 依赖
cp frontend/.env.template frontend/.env
pnpm dev                        # 前端（端口 2048）
pnpm dev:backend                # 后端 API
pnpm test
pnpm lint
```
