# 26.mock-interview-space 训练资料阶段 AI 知识补齐清单

## 1. 这份文档解决什么问题

这份文档不是通用 AI 学习路线，也不是完整大模型工程路线，而是专门服务于当前 `mock-interview-space`（模拟面试星际页面）第二阶段开发：

`训练资料真实接入 -> 题目与问答受资料影响 -> 报告与续练受资料影响`

目标很明确：

1. 告诉你当前阶段到底要补哪些知识。
2. 告诉你哪些技术现在不用急着学。
3. 告诉你这些知识分别会落到当前项目的哪一层。
4. 避免你还没把训练资料链路做实，就先掉进重型 AI 基础设施学习里。

---

## 2. 先说结论：这一阶段不要求先学 LangChain

你目前**不需要先掌握 `LangChain`（大语言模型应用编排框架）**，也不需要先搭完整 `RAG`（检索增强生成）系统。

当前阶段真正要补的是：

1. **怎么把资料结构化后喂给 LLM（大语言模型）。**
2. **怎么设计提示词，让题目、追问、报告真正受资料影响。**
3. **怎么组织流式请求、上下文和错误处理。**
4. **怎么区分“资料结构层”“提示词层”“状态层”“展示层”。**

一句话说：

**你现在先学“如何把资料接进 LLM 调用链”，而不是先学“怎么搭知识库平台”。**

---

## 3. 当前阶段真正会用到的技术栈

### 3.1 前端应用层

这是你当前项目最直接的主战场。

需要继续熟悉：

1. `Vue 3`（前端框架）
2. `TypeScript`（类型系统）
3. `Composable`（组合函数模式）
4. `Pinia`（状态管理，如果后续这部分继续复用）
5. 本项目已有的持久化和场景状态组织方式

这层负责：

1. 当前资料选择
2. 当前训练上下文保存
3. 资料状态恢复
4. 报告与专项训练的状态承接
5. 场景切换时上下文不打架

### 3.2 LLM 接入层

这是这阶段最值得补的 AI 能力，不需要很重，但要学扎实。

需要掌握：

1. `Prompt`（提示词）设计
2. 上下文组织
3. 请求体结构设计
4. `SSE`（服务端推送事件）/ 流式响应处理
5. 错误处理与中断恢复

这层负责：

1. 把资料摘要、资料标签、资料片段带进请求
2. 让 LLM 围绕资料生成题目、追问、报告
3. 控制输出风格和边界

### 3.3 轻量资料处理层

这层现在会用到，但还不需要上复杂知识库技术。

需要掌握：

1. 文本清洗
2. 资料摘要字段设计
3. 片段截取
4. 标签化
5. 主题归类

这层负责：

1. 让资料可结构化保存
2. 让资料可被题目和报告消费
3. 让资料在刷新、报告、历史中有稳定引用

### 3.4 持久化与恢复层

这是当前项目里很关键的一层，因为你现在不是做一次性问答，而是在做训练闭环。

需要掌握：

1. 当前资料引用保存
2. 当前训练轮次保存
3. 当前报告摘要保存
4. 当前专项训练计划保存
5. 历史训练记录与资料的关联

这层负责：

1. 刷新后还能恢复训练上下文
2. 报告和续练知道来自哪份资料
3. 历史记录能回看到资料维度

## 4. 这一阶段最该补的知识模块

下面按优先级排序。

### 4.1 第一优先级：Prompt 设计

这是你现在最该补的。

你要学会的不是“写很厉害的提示词”，而是：

1. 怎么让模型知道当前训练目标是什么。
2. 怎么让模型知道当前资料是什么。
3. 怎么限制模型围绕资料提问，而不是跑偏成泛问答。
4. 怎么让报告输出能引用资料上下文，而不是只给通用评价。

当前最需要掌握的能力：

1. **角色约束**
   例如：让模型扮演面试官，而不是知识助手。
2. **任务边界约束**
   例如：只围绕资料内容和训练主题提问。
3. **上下文注入**
   例如：把 `summary`（摘要）、`tags`（标签）、`topic`（主题）、片段一起放进请求。
4. **输出格式约束**
   例如：明确要求返回追问、反馈、建议时的结构。

你真正要补的是：

**如何写“稳定、可控、可解释”的训练型提示词。**

---

### 4.2 第二优先级：上下文设计

很多人以为 AI 项目的关键是模型本身，其实当前阶段更关键的是：

**你到底把什么资料喂给模型。**

你要补的知识点：

1. 什么适合放在全局系统提示里。
2. 什么适合放在本轮请求上下文里。
3. 什么应该放资料摘要。
4. 什么应该放资料原文片段。
5. 什么不该放，避免上下文太长、太乱。

当前项目里，你至少要会设计下面这些字段的用途：

1. `name`（资料名）
2. `type`（资料类型）
3. `topicKeys`（资料主题）
4. `tags`（资料标签）
5. `summary`（资料摘要）
6. `rawText`（资料正文）
7. `selectedSnippet`（当前选中的资料片段，如果后面需要）

你要学会的是：

**不是把整份资料一股脑扔给模型，而是有层次地组织资料上下文。**

---

### 4.3 第三优先级：SSE 流式接口与请求链路

你当前项目已经在用流式返回，所以这块要学，但重点是工程落地，不是概念百科。

要补的知识点：

1. `SSE`（服务端推送事件）是什么
2. 流式文本是怎么逐段返回的
3. 前端怎么消费流式分片
4. 中途停止怎么处理
5. 失败怎么兜底
6. 什么状态该保留、什么状态该回滚

你在项目里最该关心的是：

1. 资料上下文如何进入流式请求
2. 停止生成时当前 session（训练轮次）不能坏
3. 真实流和本地模拟流要走同一套稳定状态边界

也就是说，你要补的是：

**AI 请求链怎么和前端状态链对齐。**

---

### 4.4 第四优先级：结构化输出与类型约束

如果后面报告和续练要更稳定，模型输出不能永远只是一大段自然语言。

你现在可以开始补这类意识：

1. 哪些内容适合继续纯文本输出
2. 哪些内容适合结构化字段输出
3. 怎样让前端消费更稳定

例如后面这些就适合逐步结构化：

1. `weaknessTags`（弱项标签）
2. `focusArea`（补练重点）
3. `suggestedFocus`（建议项）
4. `summaryHeadline`（摘要标题）
5. `summaryBody`（摘要正文）

你不用一开始就搞严格的 JSON（结构化数据格式） 生成平台，但至少要开始建立这个判断：

**能结构化的业务信号，不要永远埋在自然语言里。**

---

### 4.5 第五优先级：轻量资料预处理

当前阶段不需要复杂文档工程，但至少要知道资料进入模型前要做什么预处理。

要补的知识点：

1. 文本清洗
2. 去掉噪声
3. 控制长度
4. 提前摘要
5. 提取关键标签

因为真实项目里最常见的问题不是模型不聪明，而是：

**喂进去的资料太脏、太散、太长。**

---

## 5. 这一阶段暂时不用急着补的知识

下面这些以后有用，但当前阶段不是前置条件。

### 5.1 LangChain

以后在这些场景会有用：

1. 多步骤链式调用
2. 多模型编排
3. 工具调用
4. 检索与生成组合
5. 标准化 Agent（智能体）流程

但你现在还没到这个复杂度。

### 5.2 向量数据库

以后在这些场景会有用：

1. 资料很多
2. 不能整份带上下文
3. 需要从大量切片里检索最相关内容

但你现在先把资料结构和轻量上下文链做好更重要。

### 5.3 Embedding

以后在语义检索、相似度匹配里有用。

但当前阶段先用：

1. 标签
2. 主题
3. 资料类型
4. 摘要片段

就足够推进。

### 5.4 完整 RAG

以后会是很重要的能力，但不是你当前项目的第一阻塞点。

你现在真正的阻塞点是：

**资料还没稳定成为训练输入。**

---

## 6. 建议学习顺序

建议你按下面顺序补，不要反过来。

### 第一阶段：先补能直接开发的知识

1. `Prompt`（提示词）设计
2. 上下文组织
3. SSE 流式请求与响应
4. 结构化输出意识

### 第二阶段：再补资料处理能力

1. 摘要设计
2. 文本截断与片段选择
3. 标签和主题抽取
4. 资料结构建模

### 第三阶段：再看要不要上更重的 AI 基础设施

1. LangChain
2. Embedding
3. 向量检索
4. RAG

这个顺序的核心逻辑是：

**先把“能用”做出来，再学“更高级的做法”。**

---

## 7. 结合当前项目，你最应该看的具体知识点

如果你现在只想抓最有用的一小段，那就盯住这几个：

1. **提示词怎么写才能让 AI 围绕资料提问**
2. **请求里怎么带资料摘要和资料片段**
3. **前端怎么处理流式返回、中断和报错**
4. **哪些 AI 输出要保留成结构化字段**
5. **资料状态怎么在训练、报告、续练之间传递**

只要你把这五件事学明白，当前第二阶段就已经够用了。

---

## 8. 你现在可以暂时不焦虑的事

你现在还不用因为“不会 LangChain”就觉得卡住。

因为当前阶段真正需要的不是“会不会一整套 AI 框架”，而是：

1. 会不会把资料组织清楚
2. 会不会把资料喂给模型
3. 会不会让模型围绕资料稳定输出
4. 会不会把输出接回前端状态链

这四件事做好了，后面再学 LangChain 会更顺。

---

## 9. 一句话结论

你当前阶段最该补的不是 `LangChain`（大语言模型应用编排框架），而是：

**Prompt 设计、上下文组织、流式请求链路、轻量资料处理、结构化输出意识。**

等训练资料真正进入训练闭环后，再决定是否升级到更重的 `RAG`（检索增强生成）或知识库技术栈。
