## 2026-06-04 恢复 docs 并重新纳入版本库

- **现象**：`docs/` 下大量 `.md` 在合并 `main` 或拉取后从磁盘消失；仅残留少量未进库的本地面经类文件。
- **根因（仓库）**：提交 `cfaef5e` 将约 51 个 `docs/*.md` 从 Git 删除，并在 `.gitignore` 写入 `/docs`，设计为「仅本地、不推送」。此后任意 `checkout`/`merge`/`pull` 到含该提交的 `main`，Git 会按树删除工作区里曾被跟踪的文件。
- **为何像丢了两次**：① 第一次：拉到 `cfaef5e` 后文件被删；② 第二次常见情况：手动从 `8cb195a` 恢复或 IDE 仍缓存旧路径，以为还在，再次 `pull`/`merge`/`checkout main` 或换机 `clone` 时仍无远程 `docs`（因已 ignore + 未提交），再次落空。并非两次独立删库提交，而是**同一策略 + 未纳入远程**的重复暴露。
- **修复**：从 `8cb195a` 恢复 `docs/`；删除 `.gitignore` 的 `/docs`；本地面经类 `.md` 与历史文档合并保留（约 60 个文件）。**禁止**再次将整目录 `docs` 加入 `.gitignore` 或 `git rm --cached docs` 而不团队确认。
- **验证**：`docs/开发记录.md`、`docs/目录说明.md` 存在；`git check-ignore -v docs/开发记录.md` 应无输出。

## 2026-05-26 修复补练题池已生成但预览 0 题

- **现象**：接口 200、角标「已生成 5 题」，预览区「共 0 题」。
- **根因**：预览组卷只读 `report.practicePlan`；远程报告合并后可能无 `practicePlan`，题池 `planSnapshot` 未参与预览。
- **修复**：预览/开练优先 `report.practicePlan ?? pool.planSnapshot`；结束面试合并远程报告时保留本地 `practicePlan` 与 `sourceDocumentExcerpt`。
